ALTERA 管理画面 / 画像・演出差し替え

この admin.html で保存した設定は、同じブラウザの localStorage に保存されます。ゲーム側の index.v6.admin-linked.html は同じキーを読んで、背景・キャラ・モンスター・アイテム・スキル画像と、スキル演出パラメータを反映します。
URL / 相対パス入力に加えて、各画像枠へ ドラッグ&ドロップ でも設定できます。例: assets/player_idle.png または https://...

MENU

概要

まずは画像URLや相対パスを入れて保存してください。ゲームを別タブで開いていれば、保存後に再読込すると反映されます。

おすすめ運用
1. まず背景・プレイヤー・モンスターだけ入れる
2. 次にスキル画像を追加する
3. 最後にスキル演出パラメータを微調整する

スキル演出は preset / color / radius / ringCount / beamWidth / sizeMul / width をこの画面から編集できます。

Supabase Storage アップロード設定

ドラッグ&ドロップした画像を Supabase Storage にアップロードして、公開URLとして保存できます。ここで保存した Supabase URL / ANON KEY は、Storage だけでなく ゲーム本体の DB / Realtime Chat / Presence にも自動利用されます。

おすすめは Supabase Storage です。画像数が増えても localStorage 容量を圧迫しにくくなります。
公開バケット名。下の SQL ファイルで altera-assets を作る想定です。
例: altera を指定すると altera/player/idle-....png のように保存されます。
接続テストでは Storage への接続とバケット参照を確認します。アップロード一覧更新で現在の保存先を走査します。
公開URL方式を使う場合は、先に Supabase 側で公開バケットとポリシーを作成してください。今回一緒に SQL ファイルも用意します。

一括ドラッグ&ドロップ投入

複数画像をまとめて落とすと、ファイル名から自動で対象欄を判定して反映します。

backgrounds_sky.png / parallax_far.png / drop_default.png のような名前にも対応します。

簡易ゲームプレビュー

保存した設定を実ゲーム表示で確認するための埋め込みプレビューです。保存時に自動で再読込されます。

このプレビューは index.v6.admin-linked.html をそのまま表示します。ローカル保存後に自動で再読込するため、背景・プレイヤー・モンスター・スキル画像の見え方を管理画面の中で確認できます。
ゲーム開始前のログイン画面やUIの見え方も確認できます。プレイ中の見え方を確かめたい場合は「別タブで開く」も使えます。

JSON

設定をまとめてバックアップ・共有したいときに使います。

JSONを貼り付けて「JSON読込」を押すと反映されます。現在設定の確認にも使えます。
